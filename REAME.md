
# Лабораторная работа №35. Подключаем фронтенд к API: доска мемов

**Выполнил:** SDA
**Группа:** ISP-232

## Описание
Проект представляет собой веб-приложение "Доска мемов", состоящее из backend-части на ASP.NET Core Web API и frontend-части на чистом HTML/CSS/JavaScript. Приложение позволяет просматривать список мемов, добавлять новые и удалять существующие без перезагрузки страницы.

## Инструкция по запуску

### 1. Запуск Backend (API)
Откройте терминал в папке проекта `MemesApi` и выполните команду:
```bash
cd MemesApi
dotnet run
```
API будет доступно по адресу: `http://localhost:5117`

### 2. Запуск Frontend
Откройте файл `frontend/index.html` в VS Code и запустите через расширение **Live Server**.


## Изученные концепции

| Концепция | Где используется |
| :--- | :--- |
| `fetch(url)` | GET-запрос к API для получения списка мемов |
| `fetch(url, { method, headers, body })` | POST и DELETE запросы для управления данными |
| `response.ok` | Проверка успешности ответа от сервера (статус 200-299) |
| `response.json()` | Чтение JSON-данных из тела ответа |
| `JSON.stringify(...)` | Преобразование объекта JS в JSON-строку для отправки на сервер |
| `async / await` | Асинхронное ожидание ответа от сервера |
| `try / catch / finally` | Обработка ошибок при fetch и управление состоянием интерфейса |
| `CORS` | Механизм безопасности, блокирующий запросы между разными портами |
| `AddCors + UseCors` | Включение CORS в ASP.NET Core (два обязательных шага настройки) |
| `API_URL` — константа | Хранение адреса сервера в одном месте для удобства изменения |
| `api.js` — отдельный файл | Вынос всех fetch-запросов в сервисный слой отдельно от логики страницы |
| `button.disabled = true` | Блокировка кнопки во время выполнения запроса для предотвращения дубликатов |
| `DevTools → Network` | Инструмент для анализа запросов, ответов и заголовков CORS |
